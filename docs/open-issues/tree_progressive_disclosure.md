# ツリーの段階的展開（折りたたみ）が未実装

## 概要
05-ui-ux.md で定義されている段階的展開機能が未実装。ノード数が増加した際のパフォーマンスと UX に影響する。

## specs の定義（05-ui-ux.md）
- **初期表示:** ルートノードから深さ3まで + 各ブランチの head のみ表示
- **折りたたみ:** 中間ノードは「...（N個のノード）」として折りたたむ
- **展開:** 折りたたまれた部分をクリックすると展開
- **想定上限:** 1会話あたり1,000ノードを想定上限とし、超える場合はコンテキストモード `minimal` をデフォルトに切り替える提案を表示

## 現在の実装
- `components/tree/tree-view.tsx` で全ノードを一括描画
- ELK.js で全ノードのレイアウトを計算
- `onlyRenderVisibleElements` は未設定（React Flow のビューポート最適化未使用）

## 影響範囲
- ノード数が数十個程度では問題なし
- 100+ ノードで React Flow の描画パフォーマンスが低下する可能性
- 1,000+ ノードで ELK.js のレイアウト計算が遅延する可能性

## 修正方法
1. ノード数に基づいて折りたたみ対象を決定するロジックを `tree-view.tsx` に追加
2. 「...（N個のノード）」を表示する折りたたみノードコンポーネントを新規作成
3. 折りたたみノードのクリックで展開する状態管理を追加
4. React Flow の `onlyRenderVisibleElements` を有効化

- [ ] パフォーマンスに問題が出た時点で実装
- [ ] 1,000 ノード超過時の `minimal` モード提案 UI を追加
