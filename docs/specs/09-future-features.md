# 09 - 将来機能

## 概要

MVPには含まないが、将来的に実装を検討する機能の定義。
MVPの設計段階で拡張性を確保しておく対象。

## 共同編集（リアルタイム同期）

### 概要
複数ユーザーが同じ会話ツリーでリアルタイムに共同作業できる機能。

### 想定される機能
- 共有された会話ツリーで各ユーザーが別ブランチを並行作業
- 他ユーザーのブランチに途中参加
- ノードへのコメント・リアクション（GitHub PR レビュー的）
- マージによる各自のブランチ結論の統合

### 技術的方針
- クライアント↔サーバー: WebSocket（Cloud Run 対応済み）
- サーバー間のイベント伝播: Cloud Pub/Sub（複数 Cloud Run インスタンス間の同期）
- 競合解決: 楽観的ロック（MVPレベル）→ 将来的に CRDTs (Yjs) を検討

### MVPでの拡張性確保
- `Node.created_by` で操作者を記録済み
- ノードの CRUD をイベントとして扱える設計にしておく
- API は REST で構築しつつ、WebSocket エンドポイントを後から追加可能な構造

## スター（ブックマーク）機能

### 概要
他人の公開リポジトリをブックマークする機能。GitHubのスターに相当。

### データモデル
```sql
CREATE TABLE star (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES "user"(id),
  repository_id UUID REFERENCES repository(id),
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(user_id, repository_id)
);
```

### 機能
- リポジトリにスターを付ける / 外す
- 自分がスターしたリポジトリ一覧
- リポジトリのスター数表示
- スター数によるリポジトリのランキング・ソート

## 会話エクスポート

### 概要
会話を外部で利用可能な形式で出力する機能。

### 出力形式
- **Markdown:** ブランチ単位の会話をMarkdown形式で出力
- **JSON:** 全ツリー構造を含むJSON出力（インポート/バックアップ用）
- **要約テキスト:** AIで会話全体を要約した短いテキスト出力

### 出力範囲
- 単一ブランチ
- 全ブランチ（ツリー全体）
- 選択したノード範囲

## フィルター・検索機能

### 概要
会話やリポジトリを効率的に探すための検索・フィルター機能。

### 検索対象
- 会話・リポジトリのタイトル
- ノードのメッセージ内容（全文検索）
- ブランチ名

### フィルター条件
- 日付範囲
- 使用したAIモデル
- ブランチ数
- 公開範囲
- タグ（将来的にタグ機能を追加する場合）

### 技術的考慮
- PostgreSQL の全文検索（`tsvector` / `tsquery`）で実現可能
- 規模が大きくなれば Elasticsearch 等の導入を検討
