# 05 - UI/UX 設計方針

## 概要

非エンジニアでもGit的操作を直感的に行えるGUIを設計する。
ノード・マインドマップ形式で思考を可視化し、革新的なUIを目指す。

## 画面構成

### メイン画面（会話画面） `/conversation/:id`

アプリの中核となる画面。AIチャットとツリー可視化を統合。

```
┌─────────────────────────────────────────────────────────┐
│  ヘッダー: 会話タイトル | ブランチ選択 | アクション      │
├───────────────────────┬─────────────────────────────────┤
│                       │                                 │
│   ツリービュー         │   チャットビュー                │
│   (React Flow)        │   (会話のメッセージ一覧)        │
│                       │                                 │
│   ノードをクリック     │   選択中ブランチの会話が        │
│   → チャットビューに   │   時系列で表示される            │
│   反映                │                                 │
│                       │                                 │
│                       ├─────────────────────────────────┤
│                       │   メッセージ入力                 │
│                       │   [モデル選択] [送信]            │
└───────────────────────┴─────────────────────────────────┘
```

**レイアウト:**
- 左パネル: ツリービュー（React Flow + ELK.js による自動レイアウト）
- 右パネル: チャットビュー（選択中ブランチの会話表示 + メッセージ入力）
- パネルサイズはリサイズ可能（ドラッグ）
- ツリービューの全画面表示モード切替可能

## ツリービュー設計（React Flow + ELK.js）

### ノードの表示

各ノードはReact Flowのカスタムノードとして描画:

```
┌──────────────────────────┐
│ 👤 ユーザーメッセージ概要 │  ← テキストは先頭40文字程度に省略
│ 🤖 AI応答概要            │
│ ──────────────────────── │
│ 12:34 | gemini-1.5-pro   │  ← タイムスタンプとモデル名
└──────────────────────────┘
```

- ノードの色はブランチごとに異なる配色
- アクティブノード（現在の会話位置）は強調表示（太枠 + グロー効果）
- 要約ノード（merge）は破線ボーダー + マージ元ブランチへのリンク表示
- **孤立ノード**（reset で到達不能になったノード）は**半透明（opacity: 0.4）** で表示。右クリックから「新しい分岐を作成」で復帰可能

### エッジ（ノード間の接続線）

- 親子関係を矢印で表現
- merge の参照リンクは破線で表示
- cherry-pick の参照は点線で表示

### レイアウト（ELK.js）

- ELK.js の `layered` アルゴリズムでツリーを上→下 or 左→右に自動配置
- ユーザーがノードをドラッグして手動配置も可能
- ズーム・パン操作対応
- ミニマップ表示

## Git操作のUI表現

Git用語をそのまま使いつつ、非エンジニアにも分かるように補足:

| Git操作 | UIラベル | 補足テキスト |
|---------|---------|-------------|
| branch | 新しい分岐を作成 | 「ここから別の話題を探索します」 |
| switch | ブランチを切り替え | 「別の話題に移動します」 |
| merge | 会話を統合 | 「別の話題の結論をこちらに取り込みます」 |
| reset | ここまで戻す | 「この時点から会話をやり直します」 |
| cherry-pick | このノードを取り込む | 「この会話を今の話題にコピーします」 |
| diff | ブランチを比較 | 「二つの話題の違いを確認します」 |
| push | リポジトリに保存 | 「この会話を保存・公開します」 |
| clone | コピーして使う | 「この会話をコピーして自分の会話にします」 |

## 操作のアクセス方法

### コンテキストメニュー（右クリック）

ノードを右クリックした際のメニュー:
- 新しい分岐を作成（branch）
- ここまで戻す（reset）
- このノードを取り込む（cherry-pick）※別ブランチのノードの場合

### ブランチ操作パネル

ヘッダー領域のブランチ選択ドロップダウン:
- 現在のブランチ名表示
- ブランチ切替（switch）
- ブランチ比較（diff）
- 会話を統合（merge）

### コマンドパレット（パワーユーザー向け）

`Ctrl+K` で起動するコマンドパレット:
- 全操作をテキスト検索で実行可能
- エンジニアユーザーが素早く操作するため

## レスポンシブ対応

| 画面サイズ | レイアウト |
|-----------|-----------|
| デスクトップ（1024px以上） | 左右分割（ツリー + チャット） |
| タブレット（768-1023px） | タブ切替（ツリー / チャット） |
| モバイル（767px以下） | チャット中心、ツリーはドロワーで表示 |

## デザイン原則

1. **情報の段階的開示:** 基本操作は目立つ場所に、高度な操作はコンテキストメニューやコマンドパレットに
2. **視覚的フィードバック:** 操作の結果をツリーのアニメーションで即座に反映
3. **ツールチップ:** Git用語には常に分かりやすい説明をツールチップで表示
4. **安全な操作:** 破壊的操作（reset, ブランチ削除）には確認ダイアログを表示。reset は孤立ノードを残すため実質的に復帰可能であり、明示的なアンドゥ機能は MVP では不要
