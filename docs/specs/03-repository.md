# 03 - GitHub的機能要件（リポジトリ管理）

## 概要

会話ツリーを「リポジトリ」として保存・管理・共有する機能。
ユーザーの過去のAI対話を構造的に整理し、見返しやすくすることが目的。

## リポジトリの基本概念

```
Conversation（ローカル）  ──push──>  Repository（リモート）
                          <──clone──
```

- **Conversation:** ユーザーが実際にAIと対話する作業領域（ローカルリポジトリに相当）
- **Repository:** push された会話のスナップショットを保存する場所（リモートリポジトリに相当）

## 機能要件

### リポジトリ作成

- ユーザーが明示的にリポジトリを作成する
- 作成時に以下を指定:
  - タイトル
  - 説明（任意）
  - 公開範囲（private / public / limited_access）

### Push（会話の保存）

**操作:** Conversation の内容を Repository に保存する。

- **全ブランチpush:** 会話内の全ブランチとノードをリポジトリに保存
- **選択的push:** 特定のブランチのみを選択してpush
  - 選択したブランチの head からルートまでのパス上の全ノードを `RepositoryNode` テーブルにコピー
- **データ独立性:** Push 時にノードを物理コピーするため、元の Conversation を削除してもリポジトリのデータは保持される
- push 後にローカルで会話を続けても Repository には影響しない
- 同じリポジトリに対して再push可能（該当ブランチの RepositoryNode を全置換）

**UI:**
1. Conversation 画面から「リポジトリにpush」ボタン
2. push先リポジトリを選択（既存 or 新規作成）
3. push するブランチを選択（チェックボックス）
4. push 実行

### 公開範囲

| 範囲 | 説明 |
|------|------|
| `private` | 所有者のみアクセス可能 |
| `public` | 全ユーザーが閲覧可能 |
| `limited_access` | 所有者が指定したユーザーのみアクセス可能 |

- `limited_access` の場合、招待メカニズムが必要（メールアドレス or ユーザーID で招待）
- 公開範囲は作成後に変更可能

### リポジトリ一覧・詳細

**一覧画面:**
- ユーザーの所有リポジトリ一覧
- 公開リポジトリの探索（Explore）
- ソート: 更新日時、作成日時、タイトル
- フィルター: 公開範囲、所有者

**詳細画面:**
- リポジトリのメタ情報（タイトル、説明、公開範囲、作成日時）
- ブランチ一覧
- ツリー構造の可視化（React Flow）
- 各ノードの会話内容の閲覧

### Clone（リポジトリのコピー）

- 他人の public リポジトリや、limited_access で招待されたリポジトリを clone できる
- clone すると新しい Conversation が作成され、元リポジトリへの参照を保持
- clone した会話を自分のリポジトリとして保存も可能

## 画面構成（概要）

```
/dashboard              ユーザーダッシュボード
/dashboard/conversations  会話一覧（ローカル）
/dashboard/repositories   リポジトリ一覧
/explore                公開リポジトリ探索
/repository/:id         リポジトリ詳細
/conversation/:id       会話画面（メインのチャット＋ツリーUI）
```

## 操作の優先度

| 機能 | 優先度 | MVP必須 |
|------|--------|---------|
| リポジトリ作成 | 高 | Yes |
| Push（全ブランチ） | 高 | Yes |
| Push（選択的） | 中 | Yes |
| 公開範囲設定（private/public） | 高 | Yes |
| 公開範囲設定（limited_access） | 低 | No |
| リポジトリ一覧 | 高 | Yes |
| リポジトリ詳細・ツリー表示 | 高 | Yes |
| Explore（公開リポジトリ探索） | 低 | No |
| Clone | 低 | No（MVP後） |
